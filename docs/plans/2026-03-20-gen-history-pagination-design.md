# 生图历史分页加载与清理优化设计

## 问题背景

用户反馈：生图太多后访问「历史」界面，网页直接崩溃，无法清理。

### 根因分析

1. `localHistory.getAll()` 一次性从 IndexedDB 加载所有记录到内存
2. GenHistory 组件使用 `items.map()` 渲染所有图片 DOM 节点
3. 图片以 base64 格式存储，每张约 2-5MB
4. 当历史记录达到几百上千张时，浏览器内存溢出崩溃

### IndexedDB 存储机制

- 数据存储在磁盘上，不主动读取时不占用内存
- 但调用 `getAll()` 时，浏览器会把数据从磁盘读取到 JavaScript 内存
- 这个「读取到内存」的过程才是问题所在

## 解决方案

采用 **方案 C：分页 + 缓存优化**

### 核心思路

1. 分页加载：每次只加载 20 条记录
2. 缓存优化：最多保留 3 页数据（60 条），旧页自动卸载
3. 清理增强：新增按天数/数量清理功能

## 数据流设计

### 分页加载流程

```
用户打开历史页面
       ↓
GenHistory 初始化
       ↓
调用 localHistory.getCount() → 获取总数
       ↓
调用 localHistory.getPage(0, 20) → 加载第一页
       ↓
渲染 20 张图片
       ↓
用户点击「加载更多」
       ↓
调用 localHistory.getPage(1, 20) → 加载第二页
       ↓
items = [...旧数据, ...新数据] (最多保留 60 条)
       ↓
如果 items.length > 60，卸载最早的数据
```

### 清理流程

```
用户点击「清理」按钮
       ↓
显示清理菜单（3 个选项）
       ↓
┌─────────────────────────────────────────────────┐
│  选项 1: 清空全部                                │
│  - 直接调用 localHistory.clear()                 │
│  - 清空 items 状态                               │
└─────────────────────────────────────────────────┘
       ↓
┌─────────────────────────────────────────────────┐
│  选项 2: 删除 X 天前的                           │
│  - 用户输入天数（默认 7 天）                      │
│  - 调用 localHistory.countOlderThan(days)        │
│  - 显示「将删除 N 张图片」                        │
│  - 用户确认后调用 deleteOlderThan(days)          │
│  - 重新加载第一页                                │
└─────────────────────────────────────────────────┘
       ↓
┌─────────────────────────────────────────────────┐
│  选项 3: 只保留最近 N 张                         │
│  - 用户输入数量（默认 100）                       │
│  - 调用 localHistory.getCount()                  │
│  - 计算将删除的数量：totalCount - N              │
│  - 显示「将删除 M 张图片」                        │
│  - 用户确认后调用 keepOnly(N)                    │
│  - 重新加载第一页                                │
└─────────────────────────────────────────────────┘
```

## 技术实现要点

### localHistory 新增方法

```typescript
// 分页查询
async getPage(page: number, pageSize: number): Promise<LocalGenItem[]>

// 获取总数
async getCount(): Promise<number>

// 删除 X 天前的记录，返回删除数量
async deleteOlderThan(days: number): Promise<number>

// 只保留最近 N 条，返回删除数量
async keepOnly(n: number): Promise<number>

// 统计 X 天前的记录数
async countOlderThan(days: number): Promise<number>
```

### GenHistory 状态管理

```typescript
const [items, setItems] = useState<LocalGenItem[]>([]);
const [page, setPage] = useState(0);
const [hasMore, setHasMore] = useState(true);
const [totalCount, setTotalCount] = useState(0);
const [isLoading, setIsLoading] = useState(false);
```

### 内存控制策略

- 每页加载 20 条
- 最多缓存 3 页（60 条）
- 超过 60 条时，卸载最早的数据
- 内存占用：60 × 3MB ≈ 180MB（可控）

## 预期效果

1. **解决崩溃问题**：不再一次性加载所有数据
2. **内存可控**：最多占用约 180MB 内存
3. **用户体验**：可以浏览所有历史，支持加载更多
4. **清理增强**：支持按天数/数量清理，操作前显示预估数量
